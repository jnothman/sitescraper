I'm not sure why you put the files in /root/Desktop, I think this will restrict the use to root only due to root ownership of that directory.

I installed ( as root ) to /usr/lib/firefox. I still have a /usr/lib/firefox2.0.0.16 directory with that version in it.

I created a desktop icon, and pointed it to the correct path and executable. I also set a working directory. Works like a charm, I still have the old version installed.

I suspect the command you used returns errors because of the place and path specified. My icon calls firefox with the command '/usr/lib/firefox/firefox'

You do not have to install to /usr/lib if you don't want to, there are other places you can put it. I would stay away from the /root dir though...

well u mean to say i can not install FF3
on RHEL4?????
As far as I know, you should be able to install FF3 to RHEL4.

There are three was to install software to RHEL4.

1. Use the package manager. The success here will depend on the level of packages available for the distro. The older the distro is, the support goes away. New releases of software may not be available as a package.

2. Download the source and compile and install the software. This means the source has to be available as a package, from somewhere. Sourceforge and Freshmeat are the two very large places to get source from. I have never done this with FF. I don't think the source is made available for FF.

3. Mozilla releases FF as a binary package. This means they have compiled it, and they provide a compressed archive with all the files you need to install it and run it.

To do the FF installation go to this link, Mozilla has created a linux how-to.

http://support.mozilla.com/en-US/kb/...ackage_manager

The page also handles what to do if the wrong version starts. I hope this helps clear up any confusion.
